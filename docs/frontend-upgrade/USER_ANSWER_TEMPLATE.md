# User Answer Template for Frontend Upgrade Workflow

Paste or adapt this when the workflow asks what improvements you want.

```txt
Approve defaults, with these preferences:

1. Redesign intensity
I want a substantial redesign within the current structure. Do not rebuild from scratch unless absolutely necessary.

2. Main screen priority
Prioritize the main user task/workspace. The user should instantly understand what to do next.

3. Product feeling
Premium, calm, professional, product-specific, and polished. Avoid generic SaaS dashboard vibes and futuristic AI clichés.

4. Navigation
Simplify navigation if the audit shows it is confusing, but preserve routes and backend-connected flows unless the plan justifies a change.

5. Preserve
Preserve backend contracts, existing working flows, route semantics, and any product constraints.

6. Replace/improve
Replace or heavily improve generic cards, naked textareas, weak buttons, default-looking layouts, poor empty/loading/error states, and inconsistent spacing/typography.

7. States and recovery
Invest strongly in empty, loading, saving, error, retry, unavailable, and stale/resume states. These should feel designed, not bolted on.

8. Responsive
Desktop should be the showcase, but mobile must be clean, readable, and without overlap.

9. Risk tolerance
Medium. I want a visible quality jump without destabilizing the product.

10. Acceptance bar
The final frontend should feel like a serious, professional MVP that I could show to early users or investors. It should not look like a patched student project.

Hard non-goals:
- No generic dashboards.
- No fake metrics.
- No unsupported features.
- No technical/debug/internal backend details in normal UI.
- No report/PDF/export controls in active-interview screens.
```
