# Product

## Register

product

## Users

SokrAI is for healthcare, operations, innovation and governance stakeholders who need to turn an early AI idea into material that can be reviewed by other people. Users may not be technical. They need clear guidance, visible safety boundaries and confidence about what to do next.

## Product Purpose

SokrAI helps a user mature an early healthcare or operational AI proposal through guided questions, structured state, resumable sessions and a final reviewable report when the backend state supports it. Success means a user can start or resume a proposal, answer the current question, understand what is still missing and prepare material for human review without seeing backend internals.

## Brand Personality

Calm, precise, trustworthy. The product should feel healthcare-appropriate, professional and minimal, without feeling empty, generic or technical.

## Anti-references

SokrAI must not look or read like a workflow debugger, backend monitor, raw API wrapper, generic AI dashboard, generic SaaS dashboard or student prototype. Normal users should not see session IDs, request IDs, raw JSON, logs, traces, source hashes, schema versions, n8n, Fastify, Ollama, PostgreSQL or workflow internals.

## Design Principles

1. Guide the user through one clear next step at a time.
2. Make state human-readable before it is machine-readable.
3. Keep safety boundaries visible without creating alarm.
4. Reserve report and export controls for the state where they are actually available.
5. Prefer calm product craft over decorative dashboard patterns.

## Accessibility & Inclusion

Target WCAG 2.2 AA for the core workflow. The interface should use clear headings, explicit labels, visible focus states, sufficient contrast, keyboard-usable controls, understandable errors and touch-friendly layouts on mobile. Motion should be restrained and respect reduced-motion preferences.
